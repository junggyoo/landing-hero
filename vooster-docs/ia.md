# LandingHero 정보 아키텍처 (IA)

## 1. 사이트맵 (사이트맵)

```
LandingHero
├── 홈 (/)
├── 히어로 생성기 (/generate)
│   ├── 단계 1: 프롬프트 입력
│   ├── 단계 2: 미리보기 선택
│   └── 단계 3: 코드 복사
├── 내 히스토리 (/history)
│   ├── 저장된 생성물 목록
│   └── 생성물 상세보기
├── 커뮤니티 (/community)
│   ├── 피드 목록
│   ├── 태그별 필터
│   └── 게시물 상세 모달
├── 요금제 (/pricing)
├── 계정 관리 (/account)
│   ├── 프로필 설정
│   ├── 구독 관리
│   └── 결제 내역
├── 인증 (/auth)
│   ├── 로그인 (/auth/login)
│   └── 회원가입 (/auth/signup)
└── 법적 고지
    ├── 이용약관 (/terms)
    └── 개인정보처리방침 (/privacy)
```

## 2. 사용자 흐름 (사용자 흐름)

### **핵심 작업: 히어로 섹션 생성**
1. 사용자가 홈페이지에서 "히어로 생성하기" 버튼 클릭
2. 비로그인 상태인 경우 로그인 모달 표시
3. 로그인 완료 후 생성기 페이지로 이동
4. 프롬프트 입력 (예: "AI PM 협업 서비스, 팀 생산성 40%↑")
5. AI 프롬프트 자동 보완 제안 확인 후 승인
6. 3가지 히어로 섹션 변형 미리보기 표시
7. 데스크톱/모바일 반응형 탭 전환하며 확인
8. 원하는 변형 선택 후 "코드 복사" 클릭
9. HTML/CSS 코드 클립보드 복사 완료
10. 자동으로 히스토리에 저장

### **핵심 작업: 커뮤니티 공유**
1. 히스토리 페이지에서 생성물 선택
2. "커뮤니티에 공유" 버튼 클릭
3. 태그 입력 및 공개 설정
4. 공유 완료 후 커뮤니티 피드에 노출
5. 다른 사용자들이 좋아요 및 북마크 가능

### **핵심 작업: 구독 업그레이드**
1. 무료 사용자가 일일 한도 초과 시 알림 표시
2. 요금제 페이지로 이동
3. Free vs Pro 기능 비교 확인
4. "Pro 구독하기" 버튼 클릭
5. Stripe 결제 페이지로 리다이렉트
6. 결제 완료 후 Pro 권한 활성화

## 3. 네비게이션 구조 (네비게이션 구조)

### **글로벌 네비게이션 바 (GNB)**
- **위치**: 상단 고정 (64px 높이)
- **구조**: 
  - 좌측: LandingHero 로고 (홈으로 이동)
  - 중앙: 주요 링크 (요금제, 커뮤니티, 문서)
  - 우측: "히어로 생성하기" CTA 버튼, 사용자 아바타
- **스크롤 상태**: 투명 → 블러 배경으로 변경

### **사이드바 네비게이션 (히스토리 페이지)**
- **위치**: 좌측 사이드바 (280px 너비)
- **구조**: 저장된 생성물 목록 (썸네일, 프롬프트, 생성 시간)
- **필터링**: 최신순, 인기순, 태그별 정렬

### **푸터**
- **구조**: 
  - 좌측: 회사 정보, 소셜 미디어 링크
  - 우측: 법적 고지 (이용약관, 개인정보처리방침)
  - 하단: 저작권 정보

## 4. 페이지 계층 구조 (페이지 계층 구조)

```
│   ├── /auth/login (Depth 2)

```

## 5. 콘텐츠 구성 (콘텐츠 구성)

| 페이지 | 주요 콘텐츠 요소 |
|---|---|
| 홈 | 히어로 섹션, 주요 기능 소개, 사용 사례, 고객 후기, CTA 버튼 |
| 히어로 생성기 | 프롬프트 입력 폼, AI 보완 제안, 미리보기 영역, 반응형 탭, 코드 출력 |
| 히스토리 | 생성물 썸네일 리스트, 검색/필터, 상세 미리보기, 코드 복사, 공유 버튼 |
| 커뮤니티 | 카드 그리드 레이아웃, 태그 필터, 좋아요/북마크, 상세 모달 |
| 요금제 | 기능 비교 테이블, 결제 버튼, FAQ, 환불 정책 |
| 계정 관리 | 프로필 폼, 구독 상태, 사용량 통계, 결제 내역 테이블 |

## 6. 인터랙션 패턴 (인터랙션 패턴)

### **모달 패턴**
- **사용 사례**: 로그인/회원가입, 커뮤니티 게시물 상세보기, 삭제 확인
- **동작**: 배경 블러 + 중앙 팝업, ESC 키로 닫기

### **토스트 알림**
- **사용 사례**: 성공/에러 메시지, 코드 복사 완료, 저장 완료
- **위치**: 우상단, 3초 후 자동 사라짐

### **스켈레톤 로딩**
- **사용 사례**: AI 생성 대기 중, 페이지 초기 로딩
- **스타일**: 실제 콘텐츠 구조를 반영한 플레이스홀더

### **무한 스크롤**
- **사용 사례**: 커뮤니티 피드, 히스토리 목록
- **구현**: 하단 도달 시 추가 콘텐츠 로드

### **드래그 앤 드롭**
- **사용 사례**: 히스토리에서 생성물 순서 변경
- **피드백**: 드래그 중 시각적 피드백 제공

## 7. URL 구조 (URL 구조)

### **일반 페이지**
- 홈: `/`
- 생성기: `/generate`
- 히스토리: `/history`
- 커뮤니티: `/community`

### **상세 페이지**
- 생성물 상세: `/history/:generationId`
- 커뮤니티 게시물: `/community/:postId`
- 계정 설정: `/account/:section`

### **쿼리 파라미터**
- 생성 단계: `/generate?step=1`
- 커뮤니티 필터: `/community?tag=saas&sort=popular`
- 검색: `/history?search=landing&page=2`

### **SEO 친화적 URL**
- 소문자 사용
- 하이픈으로 단어 구분
- 의미 있는 키워드 포함

## 8. 컴포넌트 계층 구조 (컴포넌트 계층 구조)

### **글로벌 컴포넌트**
- **Header**: 네비게이션, 로고, 사용자 메뉴
- **Footer**: 링크, 법적 고지, 소셜 미디어
- **Toast**: 알림 메시지 표시
- **Modal**: 로그인, 확인 다이얼로그
- **Loading**: 스켈레톤 UI, 스피너

### **페이지별 컴포넌트**
- **홈페이지**: HeroSection, FeatureGrid, TestimonialCarousel, CTASection
- **생성기**: PromptInput, AIEnhancer, PreviewTabs, CodeOutput
- **히스토리**: GenerationCard, FilterBar, SearchInput, DetailView
- **커뮤니티**: PostCard, TagFilter, LikeButton, BookmarkButton

### **폼 컴포넌트**
- **Input**: 텍스트 입력, 검증 상태 표시
- **Button**: Primary, Secondary, Text 버튼 변형
- **Select**: 드롭다운 선택
- **Textarea**: 프롬프트 입력용 확장 텍스트 영역

### **데이터 표시 컴포넌트**
- **Card**: 생성물, 게시물 카드
- **Badge**: 태그, 상태 표시
- **Avatar**: 사용자 프로필 이미지
- **ProgressBar**: 생성 진행률 표시

### **레이아웃 컴포넌트**
- **Container**: 최대 너비 제한 래퍼
- **Grid**: 반응형 그리드 시스템
- **Stack**: 수직/수평 스택 레이아웃
- **Sidebar**: 사이드바 레이아웃
